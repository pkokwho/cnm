# EvidenceBox 证据盒

> 把混乱材料变成可执行方案 — 社会服务赛道创意方案

当聊天记录、截图、票据、录音碎片堆积如山，你不是缺信息，而是缺把信息变成行动的能力。EvidenceBox 帮你从混乱中提取秩序。

## 核心功能

- **智能时间线** — 自动按时间顺序排列所有材料，清晰呈现事件发展脉络
- **关键摘要** — 从海量碎片中提取核心信息，生成一句话摘要和要点列表
- **待办清单** — 自动识别材料中需要跟进的事项，生成可勾选的任务列表
- **下一步建议** — 基于整理结果，给出具体的后续行动建议

## 使用流程

1. **丢入材料** — 拖拽上传聊天记录、截图、PDF、TXT，支持批量导入
2. **自动分析** — 系统自动识别时间、人物、事件、关键语句
3. **获取方案** — 瞬间获得时间线、摘要、待办清单和行动建议，一键导出 Markdown

## 技术栈

- **前端**: Next.js 16 + React 19 + TailwindCSS
- **存储**: SQLite (本地) / 内存存储 (Vercel 部署)
- **文本提取**: pdf-parse (PDF), Tesseract.js (OCR), 原生文本解析
- **分析引擎**: 纯 JS 规则引擎 — 正则日期提取 + TF-IDF 摘要 + 指令词匹配
- **国际化**: 中英双语切换

## 本地开发

```bash
npm install
npm run dev
```

打开 http://localhost:3000

## 部署

### Vercel 部署

1. Fork 本仓库
2. 在 [Vercel](https://vercel.com) 中导入该仓库
3. 自动部署，无需额外配置

> 注意：Vercel 部署使用内存存储，数据在 Serverless 函数冷启动后可能重置。本地开发使用 SQLite 持久化存储。

## 支持的文件格式

| 格式 | 说明 |
|------|------|
| TXT | 纯文本、聊天记录导出 |
| CSV | 结构化数据 |
| PDF | 文本型 PDF 自动提取，扫描型 PDF 使用 OCR |
| PNG/JPG | 截图、照片，使用 OCR 提取文字 |

## 项目结构

```
├── app/                    # Next.js App Router
│   ├── page.tsx           # 落地页
│   ├── app/               # 工作台

│   │   └── cases/[id]/    # 案件详情

│   └── views/             # 分析结果视图
├── lib/
│   ├── analyzer/          # 分析引擎（日期/摘要/待办/建议）
│   ├── i18n/              # 中英双语翻译
│   ├── processors/        # 文件处理器（PDF/OCR/文本）
│   └── store/             # 数据存储抽象层
└── public/                # 静态资源
```
